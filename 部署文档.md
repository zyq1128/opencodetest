## 快速部署

### 方式一：一键部署脚本（推荐）

创建以下批处理文件 `一键部署.bat`：

```batch
@echo off
chcp 65001 >nul
echo ========================================
echo  春保森拉天时工业AI平台 - 一键部署脚本
echo ========================================
echo.

:: 检查管理员权限
net session >nul 2>&1
if %errorlevel% neq 0 (
    echo [错误] 请以管理员权限运行此脚本！
    pause
    exit /b 1
)

:: 设置变量
set "INSTALL_DIR=C:\CBC_AIPOC"
set "PYTHON_URL=https://www.python.org/ftp/python/3.11.8/python-3.11.8-amd64.exe"
set "NODE_URL=https://nodejs.org/dist/v20.11.1/node-v20.11.1-x64.msi"

echo [1/7] 检查并安装 Python...
python --version >nul 2>&1
if %errorlevel% neq 0 (
    echo 正在下载 Python...
    powershell -Command "Invoke-WebRequest -Uri '%PYTHON_URL%' -OutFile '%TEMP%\python_installer.exe'"
    echo 正在安装 Python...
    %TEMP%\python_installer.exe /quiet InstallAllUsers=1 PrependPath=1
) else (
    echo Python 已安装
)

echo [2/7] 检查并安装 Node.js...
node --version >nul 2>&1
if %errorlevel% neq 0 (
    echo 正在下载 Node.js...
    powershell -Command "Invoke-WebRequest -Uri '%NODE_URL%' -OutFile '%TEMP%\nodejs_installer.msi'"
    echo 正在安装 Node.js...
    msiexec /i %TEMP%\nodejs_installer.msi /quiet /norestart
) else (
    echo Node.js 已安装
)

echo [3/7] 创建安装目录...
mkdir "%INSTALL_DIR%" 2>nul

echo [4/7] 复制项目文件...
:: 假设当前目录是项目根目录
xcopy /E /I /Y "backend" "%INSTALL_DIR%\backend\" >nul
xcopy /E /I /Y "frontend" "%INSTALL_DIR%\frontend\" >nul
xcopy /E /I /Y "simulator" "%INSTALL_DIR%\simulator\" >nul
copy /Y "README.md" "%INSTALL_DIR%\" >nul

echo [5/7] 安装 Python 依赖...
cd /d "%INSTALL_DIR%\backend"
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt

echo [6/7] 安装 Node.js 依赖...
cd /d "%INSTALL_DIR%\frontend"
npm install

echo [7/7] 创建启动脚本...
(
echo @echo off
echo start "后端服务" cmd /k "cd /d %INSTALL_DIR%\backend ^&^& .venv\Scripts\activate ^&^& python main.py"
echo timeout /t 3 /nobreak ^>nul
echo start "模拟器" cmd /k "cd /d %INSTALL_DIR%\simulator ^&^& python device_simulator.py"
echo timeout /t 3 /nobreak ^>nul
echo start "前端服务" cmd /k "cd /d %INSTALL_DIR%\frontend ^&^& npm run dev"
echo timeout /t 5 /nobreak ^>nul
echo start http://localhost:5173
echo echo 所有服务已启动！
echo pause
) > "%INSTALL_DIR%\启动全部服务.bat"

echo.
echo ========================================
echo  部署完成！
echo ========================================
echo  安装目录: %INSTALL_DIR%
echo  启动脚本: %INSTALL_DIR%\启动全部服务.bat
echo.
echo  请运行启动脚本开启所有服务
echo ========================================
pause
```

## 手动详细部署步骤

如果一键部署脚本无法使用，请按以下步骤手动部署：

### 第1步：安装依赖软件

#### 1.1 安装 Python 3.11

1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.11.8 (64-bit)
3. 运行安装程序，**务必勾选** "Add Python to PATH"
4. 选择 "Customize installation" → 勾选 "Install for all users"
5. 点击 Install

验证安装：
```cmd
python --version
# 应显示 Python 3.11.8

pip --version
# 应显示 pip 版本
```

#### 1.2 安装 Node.js 20 LTS

1. 访问 https://nodejs.org/
2. 下载 Node.js 20.11.1 LTS (64-bit)
3. 运行安装程序，保持默认选项
4. 完成安装

验证安装：
```cmd
node --version
# 应显示 v20.11.1

npm --version
# 应显示 10.x.x
```

### 第2步：复制项目文件

1. 在目标电脑上创建项目目录：
```cmd
mkdir C:\CBC_AIPOC
cd C:\CBC_AIPOC
```

2. 将项目文件复制到该目录：
   - `backend/` 文件夹 → `C:\CBC_AIPOC\backend\`
   - `frontend/` 文件夹 → `C:\CBC_AIPOC\frontend\`
   - `simulator/` 文件夹 → `C:\CBC_AIPOC\simulator\`

### 第3步：安装后端依赖

1. 打开命令提示符（CMD），进入后端目录：
```cmd
cd C:\CBC_AIPOC\backend
```

2. 创建 Python 虚拟环境：
```cmd
python -m venv .venv
```

3. 激活虚拟环境：
```cmd
.venv\Scripts\activate
```

4. 安装依赖包：
```cmd
pip install -r requirements.txt
```

此过程需要 5-10 分钟，安装完成后会显示 `Successfully installed ...`

### 第4步：安装前端依赖

1. 打开新的命令提示符，进入前端目录：
```cmd
cd C:\CBC_AIPOC\frontend
```

2. 安装 npm 依赖：
```cmd
npm install
```

此过程需要 3-5 分钟，安装完成后会创建 `node_modules` 文件夹。

### 第5步：验证安装

#### 验证后端

```cmd
cd C:\CBC_AIPOC\backend
.venv\Scripts\activate
python main.py
```

成功启动后会显示：
```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

访问 http://localhost:8000/api/docs 验证 API 文档是否正常。

#### 验证前端

```cmd
cd C:\CBC_AIPOC\frontend
npm run dev
```

成功启动后会显示：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
  ➜  press h + enter to show help
```

访问 http://localhost:5173 验证前端是否正常加载。

## 故障排查

### 常见问题及解决方案

#### 1. 后端启动失败

**症状**：运行 `python main.py` 后报错

**可能原因及解决方案**：

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `'python' 不是内部或外部命令` | Python 未安装或未添加到PATH | 重新安装Python，勾选"Add to PATH" |
| `No module named 'fastapi'` | 依赖未安装 | 运行 `.venv\Scripts\activate` 后执行 `pip install -r requirements.txt` |
| `Address already in use` | 端口8000被占用 | 关闭占用8000端口的程序，或修改main.py中的端口 |
| `Permission denied` | 权限不足 | 以管理员身份运行CMD |

**检查步骤**：
```cmd
cd C:\CBC_AIPOC\backend

# 1. 检查Python
python --version

# 2. 激活虚拟环境
.venv\Scripts\activate

# 3. 检查已安装包
pip list

# 4. 重新安装依赖（如有问题）
pip install -r requirements.txt --force-reinstall
```

#### 2. 前端启动失败

**症状**：运行 `npm run dev` 后报错

**可能原因及解决方案**：

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `'npm' 不是内部或外部命令` | Node.js 未安装 | 安装Node.js 18+ LTS版本 |
| `Cannot find module 'react'` | node_modules缺失 | 运行 `npm install` |
| `Port 5173 is already in use` | 端口被占用 | 自动切换到下一个端口，或关闭占用进程 |
| `Failed to load module` | Vite缓存问题 | 删除 `node_modules/.vite` 后重试 |

**检查步骤**：
```cmd
cd C:\CBC_AIPOC\frontend

# 1. 检查Node.js
node --version
npm --version

# 2. 清理缓存并重新安装
rmdir /s /q node_modules 2>nul
del package-lock.json 2>nul
npm cache clean --force
npm install

# 3. 启动开发服务器
npm run dev
```

#### 3. 模拟器启动失败

**症状**：运行 `python device_simulator.py` 报错

**解决方案**：
```cmd
cd C:\CBC_AIPOC\simulator

# 确保使用与后端相同的Python环境
C:\CBC_AIPOC\backend\.venv\Scripts\python.exe device_simulator.py

# 或先激活虚拟环境
call C:\CBC_AIPOC\backend\.venv\Scripts\activate
python device_simulator.py
```

#### 4. 前端无法连接后端

**症状**：前端页面打开但显示"连接中"或没有数据

**排查步骤**：

1. **检查后端是否运行**：
   ```cmd
   curl http://localhost:8000/
   # 应该返回JSON数据
   ```

2. **检查WebSocket连接**：
   - 浏览器F12 → Network → WS
   - 查看 `ws://localhost:8000/ws/realtime` 连接状态
   
3. **检查Vite代理配置**：
   - 打开 `frontend/vite.config.js`
   - 确认代理设置：
   ```javascript
   proxy: {
     '/api': 'http://localhost:8000',
     '/ws': { target: 'ws://localhost:8000', ws: true }
   }
   ```

4. **检查防火墙**：
   - Windows防火墙可能阻止8000端口
   - 添加例外规则：
     ```cmd
     netsh advfirewall firewall add rule name="CBC AI Backend" dir=in action=allow protocol=tcp localport=8000,5173
     ```

#### 5. 数据库相关错误

**症状**：后端报错 `sqlite3.OperationalError` 或数据库锁定

**解决方案**：

1. **删除并重建数据库**：
   ```cmd
   cd C:\CBC_AIPOC\backend
   del industrial_data.db
   :: 重启后端服务，会自动重建数据库
   ```

2. **检查数据库权限**：
   ```cmd
   icacls industrial_data.db /grant Everyone:F
   ```

3. **避免多进程访问**：
   - 确保只有一个后端实例运行
   - 使用 `tasklist | findstr python` 检查多个Python进程

## 日常维护

### 数据备份

```batch
@echo off
set BACKUP_DIR=C:\CBC_AIPOC\backups
set DB_FILE=C:\CBC_AIPOC\backend\industrial_data.db
set TIMESTAMP=%date:~0,4%%date:~5,2%%date:~8,2%_%time:~0,2%%time:~3,2%%time:~6,2%
set TIMESTAMP=%TIMESTAMP: =0%

mkdir %BACKUP_DIR% 2>nul
copy %DB_FILE% %BACKUP_DIR%\industrial_data_%TIMESTAMP%.db

echo 备份完成: %BACKUP_DIR%\industrial_data_%TIMESTAMP%.db
```

### 日志查看

```cmd
:: 实时查看后端日志
tail -f C:\CBC_AIPOC\backend\app.log

:: 查看Windows事件日志
eventvwr.msc
:: 导航到: Windows 日志 → 应用程序
```

### 服务自动启动配置

创建 Windows 任务计划程序任务，实现开机自启：

```powershell
# 创建后端启动任务
$Action = New-ScheduledTaskAction -Execute "cmd.exe" -Argument "/c cd /d C:\CBC_AIPOC\backend && .venv\Scripts\activate && python main.py"
$Trigger = New-ScheduledTaskTrigger -AtStartup
$Settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable
Register-ScheduledTask -TaskName "CBC_AI_Backend" -Action $Action -Trigger $Trigger -Settings $Settings -User "SYSTEM" -RunLevel Highest
```
